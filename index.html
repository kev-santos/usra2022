Review: Time complexity of algorithms
June 22

Definitions:

  • Time complexity of an algorithm: $T:\mathbb{N}\to\mathbb{N}$ where $T(n)$ is the maximum number of steps in any execution of the algorithm on any input of size $n$.
  • The following are all subsets of the set of functions from $\mathbb{N}_k=\{k,k+1,k+2,...\}$ ($k\in\mathbb{N}$) to $\mathbb{R}_0^+$ (nonnegative reals)
  • $f \in O(g)$ if there exists $c\in\mathbb{R}^+$ and $n_0\in\mathbb{N}$ such that for all $n\geq n_0$, $f(n)\leq c\cdot g(n)$. $g$ is an \textbf{asymptotic upper bound} for $f$.
  • $f\in\Omega(g)$ if there exists $d\in\mathbb{R}^+$ and $m_0\in\mathbb{N}$ such that for all $n\geq m_0$, $f(n)\geq d\cdot g(n)$. $g$ is an \textbf{asymptotic lower bound} for $f$.
  • $\Theta(g) = O(g)\cap \Omega(g)$. $f\in\Theta(g)$ means $g$ is an asymptotic tight bound for $f$. (there exists $c,d\in\mathbb{R}^+$ and $n_0\in\mathbb{N}$ such that $d\cdot g(n)\leq f(n)\leq c\cdot g(n)$ for all $n\geq n_0$.)
  • $f\in o(g)$ if $\lim_{n\to\infty}f(n)/g(n)=0$.
